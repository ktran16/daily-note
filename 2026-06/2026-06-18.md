# Daily Notes - 2026-06-18

## 00:04 - dbt Sources & Seeds: Declaring Raw Data and Loading Static Files

Two foundational dbt features — **sources** and **seeds** — solve the problem of connecting your transformation layer to the outside world cleanly and traceably.

### Sources: Documenting and Referencing Raw Tables

A source is a declaration that a raw table exists in your warehouse and that dbt models should depend on it. Instead of hardcoding database/schema/table names throughout your SQL, you define them once in a YAML file and reference them with the `source()` function.

```yaml
# models/staging/_sources.yml
version: 2

sources:
  - name: stripe
    database: raw
    schema: stripe
    tables:
      - name: payments
        description: "Raw payment events from the Stripe API"
        loaded_at_field: created_at   # used for freshness checks
        freshness:
          warn_after: {count: 6, period: hour}
          error_after: {count: 24, period: hour}
        columns:
          - name: id
            tests:
              - unique
              - not_null
```

Reference it in a model with `{{ source('stripe', 'payments') }}`:

```sql
-- models/staging/stg_payments.sql
select
    id          as payment_id,
    amount,
    currency,
    created_at
from {{ source('stripe', 'payments') }}
```

This gives you:
- **Lineage** — the DAG tracks raw tables as root nodes.
- **Freshness checks** — `dbt source freshness` alerts you when upstream data is stale before you even run transforms.
- **Documentation** — column descriptions and tests live alongside the source declaration.

### Seeds: Loading Static CSV Files as Tables

Seeds let you commit small, static reference datasets as CSV files and load them into your warehouse with `dbt seed`. They're ideal for things that change rarely: country codes, account tier mappings, cost center lists.

```
seeds/
  account_tiers.csv
```

```csv
tier_id,tier_name,monthly_limit
1,Starter,1000
2,Growth,10000
3,Enterprise,999999
```

Reference it just like any other model:

```sql
-- models/marts/orders_enriched.sql
select
    o.*,
    t.tier_name
from {{ ref('orders') }} o
left join {{ ref('account_tiers') }} t
    on o.account_tier_id = t.tier_id
```

Configure seed behavior in `dbt_project.yml`:

```yaml
seeds:
  my_project:
    account_tiers:
      +schema: reference_data   # load into a specific schema
      +quote_columns: true
```

### Key Distinction

| | Sources | Seeds |
|---|---|---|
| Data lives in | Warehouse (loaded externally) | Git repo (CSV files) |
| Purpose | Reference existing raw tables | Load small static datasets |
| Command | `dbt source freshness` | `dbt seed` |
| Reference | `{{ source('name', 'table') }}` | `{{ ref('filename') }}` |

Use sources for any table loaded by an ELT tool (Fivetran, Airbyte, custom pipelines). Use seeds for lookup tables that belong in version control alongside your code.
